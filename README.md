# CzZ_EasyLan

一个简单的局域网通讯模块，Android 原生实现，零第三方依赖。

## 这是什么

这不是一个完整的 App，而是一个可以放到任何项目里使用的通讯模块。

UDP 广播发现、Socket 连接、心跳保持、数据收发 —— 全部自己实现。

## 能做什么

- 局域网内自动发现设备
- 建立点对点连接
- 稳定保持长连接
- 与 iOS / PC 数据互通（协议通用）

## 技术栈

- Android 原生 API
- 无第三方框架
- 自定义通讯协议
- 完整数据模型

## 谁可以用

- 想学局域网通讯的开发者
- 需要在项目里集成通讯功能的人
- 想移植到其他平台的人

## 架构设计思路图

```tex
┌─────────────────────────────────────────────────────────────────────────────┐
│                           设备 A                            设备 B           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【第一阶段：UDP 身份发现与认证】                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │   1. 双方协定 ──► 端口号 + 广播地址                                     │   │
│  │                  (例: 14445 / 255.255.255.255)                      │   │
│  │                              │                                       │   │
│  │   2. 广播身份消息 ──► "我是设备A，请求认证"  ──► 设备B收到              │   │
│  │                                                                     │   │
│  │   3. 回复认证消息 ──► "我是设备B，认证通过"  ──► 设备A收到              │   │
│  │                                                                     │   │
│  │   4. ✅ 初步身份认证完成                                              │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  【第二阶段：TCP 桥接通信】                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │   方式一：设备A主动发起                                               │   │
│  │   ┌──────────────┐              ┌──────────────┐                   │   │
│  │   │    设备A     │ ──TCP连接──► │    设备B     │                   │   │
│  │   │  (客户端)    │              │  (服务端)    │                   │   │
│  │   └──────────────┘              └──────────────┘                   │   │
│  │                                                                     │   │
│  │   方式二：设备B主动发起                                               │   │
│  │   ┌──────────────┐              ┌──────────────┐                   │   │
│  │   │    设备A     │ ◄──TCP连接── │    设备B     │                   │   │
│  │   │  (服务端)    │              │  (客户端)    │                   │   │
│  │   └──────────────┘              └──────────────┘                   │   │
│  │                                                                     │   │
│  │   建立后：                                                           │   │
│  │   ├── 数据互传 (JSON/文本/结构化数据)                                 │   │
│  │   └── 文件传输 (分块发送/断点续传)                                    │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
